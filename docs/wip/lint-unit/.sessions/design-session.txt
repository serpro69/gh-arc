  ---
  Section 2: Shared Runner Engine (internal/runner/)

  The runner engine is the core abstraction. It takes a list of runner
  configurations and executes them sequentially, streaming output and collecting
   results.

  Key types:

  - RunnerConfig вҖ” unified config for both lint and unit runners: name, command,
   args, workingDir, timeout, and optional fixArgs/autoFix (lint-only fields
  ignored by unit)
  - RunResult вҖ” per-runner outcome: name, status (pass/fail/error/skipped), exit
   code, duration
  - ExecutionResult вҖ” aggregate of all RunResults, overall success bool
  - Engine вҖ” the executor. Takes []RunnerConfig + EngineOptions (json mode,
  verbose, file list for lint)

  Execution flow:

  1. For each runner: print banner (> Running <name>...)
  2. Build the command: command + args (+ fixArgs if --fix) (+ file paths for
  lint runners)
  3. Execute with os/exec, streaming stdout/stderr directly to terminal
  (preserving TTY colors)
  4. Capture exit code and duration
  5. Print per-runner summary line (вң“ <name>: passed or вң— <name>: 3 issues 
  found)
  6. After all runners: print overall summary
  7. Return ExecutionResult

  --json mode: Suppresses banners and streaming output. Collects stdout/stderr
  per runner (optional). Returns JSON with execution metadata per PAL's
  recommendation вҖ” runner name, status, exit_code, duration_ms.

  File path passing: For lint, changed file paths are appended to the runner's
  args. For unit, no file paths вҖ” args run as-is.

  Timeouts: If a runner config specifies a timeout, wrap exec.CommandContext
  with a deadline. Default: no timeout (let the tool run to completion).

  unit-specific layers?

в—Ҹ Section 3: gh arc lint вҖ” Lint-Specific Layer (internal/lint/)

  This layer wraps the runner engine with lint concerns: changed-file detection,
   file filtering, and the --fix flag.

  Workflow (the LintWorkflow orchestrator):

  1. Resolve runners. Load LintConfig.Runners from config. If empty and
  MegaLinter enabled: "true" (or "auto" + Docker detected) вҶ’ MegaLinter runner
  (v2). If nothing вҶ’ print guidance message, exitВ 0.
  2. Detect changed files. Find the merge-base between HEAD and the default
  branch (git.GetMergeBase(defaultBranch, "HEAD")), then
  git.GetChangedFiles(mergeBase, "HEAD"). If --all flag, skip this and pass no
  file paths (runner lints everything).
  3. Filter files. Remove deleted files (can't lint them). Optionally filter by
  file extension if a runner declares which extensions it handles (future вҖ” not
  in v1).
  4. Convert runner configs. Map config.LintRunner вҶ’ runner.RunnerConfig,
  injecting fixArgs when --fix is active.
  5. Execute. Call runner.Engine.Run() with configs + changed file paths.
  6. Return results. Propagate ExecutionResult to the CLI layer.

  CLI flags (cmd/lint.go):
  - --fix вҖ” enable auto-fix mode (appends fixArgs to each runner)
  - --all вҖ” lint all files, not just changed ones
  - No positional args (enforced with cobra.NoArgs per the review finding)

  Exit codes:
  - 0 вҖ” all runners passed
  - 1 вҖ” one or more runners reported issues (non-zero exit)
  - The command never exits >1 for lint findings вҖ” the distinction between "lint
   issues" and "runner crash" is the runner's exit code, which is reported in
  the summary
